# OpenSorSe v0.9.1 Release Proposal

| Field | Value |
| --- | --- |
| Target release | v0.9.1 |
| Theme | Optional constrained AI controls and progressive feature visibility |
| Scope type | Focused pre-1.0 refinement and hardening |
| Depends on | Completed v0.9 historical catalog comparison and existing v0.3 Ollama boundary |

## Purpose

v0.9.1 makes the optional local AI boundary explicit, capability-specific, and impossible to reach while disabled. It also introduces an independent advanced-interface switch so ordinary scanning, results, duplicate review, catalog, search, tags, saved searches, rules, settings, and safety feedback remain approachable without discarding expert configuration.

The release adds only two AI capabilities: review-only file-rename suggestions and review-only logical folder-structure suggestions. It is not the v1.0 plugin, localization, packaging, database, semantic-search, or cross-platform release programme.

## Delivery boundary

- AI and advanced features default off and persist through the existing JSON settings service.
- AI visibility and command/service access use the same centralized feature requirements.
- Ollama remains an externally managed local service; no provider probing occurs while AI is disabled.
- Capability-specific prompt builders emit deterministic, bounded, metadata-only prompts.
- Strict application-owned parsers validate untrusted JSON before publishing immutable proposals.
- Accepting or editing a proposal records a review decision only; it never mutates the filesystem.
- Existing settings files remain loadable when the new properties are absent.

See [Specification 046](046_Optional_AI_and_Advanced_Feature_Controls.md) for the complete functional, architectural, safety, migration, test, and acceptance contract.
