## Purpose

OpenSorSe 1.0.0 builds on the safe v0.9.1 scan, Results, catalog, tag, Help, Diagnostics, and constrained Ollama boundaries. It adds a fixed Results toolbar, a right-side Duplicate View drawer, global shell feature toggles, local metadata extraction, page-aware capability-detected OCR Beta, separately gated AI document-text proposals, provenance-aware tags, local hybrid Semantic search Beta, and durable folder-structure history.

The release remains local-first and non-destructive by default. Scanning, extraction, indexing, diagrams, AI generation, and previews never change source files. A restructuring can reach the filesystem only through a separately confirmed, bounded application operation that validates the current root, rejects conflicts, records its outcome, and never overwrites or deletes.

## Feature maturity

| Area | Maturity | Boundary |
| --- | --- | --- |
| Results toolbar, Duplicate drawer, shell toggles | Stable | Presentation and settings only |
| Metadata extraction | Stable for documented bounded formats | Defensive local reads only |
| OCR | Beta | Built-in PDF rendering plus optional external local Tesseract; capability/languages are reported truthfully |
| Semantic search | Beta | Deterministic local hybrid index; quality may evolve |
| Structure history and comparison | Stable | Stored metadata and read-only diagrams |
| Restructuring application | Guarded | Explicit confirmation, no overwrite/delete, audited |
| Plugins, cloud content analysis, GPU acceleration | Deferred | Not included |
